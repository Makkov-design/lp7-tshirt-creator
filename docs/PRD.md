# PRD: "Создай свою футболку"

## 1. Контекст

Лидерская Программа N7 готовит брендированные именные футболки для 29 участников. Нужно собрать от каждого участника размер, имя, фамилию и 3 слова, с которыми человек себя ассоциирует.

Опрос должен ощущаться не как обычная анкета, а как маленький интерактивный ритуал создания собственной лидерской футболки. Участник пошагово вводит данные, сразу видит мокап будущего дизайна, подтверждает результат и наблюдает, как его футболка попадает в общую сетку участников.

## 2. Цели

- Собрать данные по всем 29 участникам без ручного хаоса в чатах.
- Сделать процесс эмоциональным, стильным и запоминающимся.
- Сразу показывать участнику персонализированный дизайн на мокапе футболки.
- Не позволять одному и тому же браузеру повторно отправить вторую футболку после завершения.
- Показывать общий прогресс сбора футболок: кто уже создал дизайн, а кто еще нет.
- Спроектировать флоу mobile-first: основной сценарий должен быть максимально удобен на телефоне.

## 3. Не цели MVP

- Не делать полноценный интернет-магазин или оплату.
- Не делать личные аккаунты и пароли для участников на первом этапе.
- Не генерировать финальные production-ready print-файлы автоматически, если это не потребуется отдельно.
- Не строить сложную админ-панель в первой версии, если достаточно таблицы данных и базового экспортного сценария.

## 4. Аудитория

Основная аудитория: 29 участников ЛП-7.

Контекст использования:
- человек в первую очередь открывает ссылку с телефона из общего чата;
- ноутбук и desktop считаются вторичными сценариями;
- проходит короткий интерактивный флоу за 1-3 минуты;
- хочет быстро понять, что именно будет на футболке;
- после отправки может вернуться и увидеть общий статус сбора.

## 5. Основная идея продукта

Название сайта: "Создай свою футболку".

Базовые элементы дизайна футболки уже заданы:
- название команды: "СОЗДАТЕЛИ";
- девиз: "СОЗДАЙ СЕБЯ, ЧТОБЫ СОЗДАТЬ МИР!";
- персональные данные участника: имя, фамилия и 3 слова-ассоциации;
- размер футболки.

Персонализация должна выглядеть как часть дизайна, а не как текст, механически наложенный поверх картинки.

## 6. Пользовательский флоу

### 6.1 Первый вход

Участник открывает сайт и видит темную минималистичную сцену с мокапом футболки и призывом начать создание.

Основные элементы:
- крупный мокап футболки;
- название "СОЗДАТЕЛИ";
- девиз команды;
- кнопка старта: "Создать футболку".

### 6.2 Шаг 1: Найти себя

Участник вводит свое имя или фамилию. Сайт ищет совпадения в заранее загруженном списке 29 участников.

После выбора участника сайт показывает его фотографию и playful-подтверждение:
- "Ооо, это ты?"
- фото участника;
- имя и фамилия из списка;
- кнопки "Да, это я" и "Нет, не я".

Если участник уже создал футболку:
- форма не открывается;
- показывается забавное предупреждение в духе "Эта футболка уже сотворена. Вторую вселенную пока не запускаем.";
- участник видит общий статус и сетку футболок.

Проверка:
- нельзя перейти дальше, пока участник не выбран и не подтвержден;
- если найдено несколько совпадений, нужно показать варианты;
- если совпадений нет, показать мягкую подсказку проверить написание.

### 6.3 Шаг 2: Размер

Участник выбирает размер футболки.

Предлагаемые размеры:
- XS
- S
- M
- L
- XL
- XXL
- 3XL

Проверка:
- размер обязателен;
- нельзя перейти дальше без выбора.

### 6.4 Шаг 3: Имя и фамилия

Имя и фамилия подтягиваются из выбранного участника. В MVP можно разрешить аккуратное редактирование отображаемого имени, если человек хочет иначе написать имя на футболке.

Поля:
- имя;
- фамилия.

Проверка:
- оба поля обязательны;
- минимальная длина каждого поля: 2 символа;
- лишние пробелы удаляются;
- регистр можно сохранять как ввел участник, либо привести к аккуратному виду на этапе дизайна.

### 6.5 Шаг 4: 3 слова

Участник вводит три слова, с которыми себя ассоциирует.

Поля:
- слово 1;
- слово 2;
- слово 3.

Проверка:
- все 3 слова обязательны;
- каждое слово желательно ограничить 20 символами, чтобы не ломать дизайн;
- если участник вводит фразу из нескольких слов, MVP может показать мягкую подсказку: "Лучше одно короткое слово".

### 6.6 Шаг 5: Предпросмотр

Участник видит финальный мокап футболки с персонализацией:
- имя и фамилия;
- 3 слова;
- название команды;
- девиз;
- выбранный размер отдельно рядом с мокапом.

Доступные действия:
- "Назад" для исправления;
- "Подтвердить дизайн" для отправки.

### 6.7 Финальная сцена

После подтверждения:
- футболка визуально "улетает" или перемещается в общую сетку;
- появляется статус: "Твоя футболка создана";
- участник видит сетку всех 29 футболок;
- для незаполненных участников отображаются карточки участников в статусе ожидания.

После перезагрузки страницы этот же браузер открывает финальную сцену. Если человек зайдет с другого устройства и выберет того же участника, сайт увидит статус в базе и не даст создать вторую футболку.

## 7. Общая сетка футболок

Сетка должна показывать прогресс по всем 29 участникам.

Для созданных футболок:
- мини-мокап футболки;
- имя и фамилия;
- 3 слова;
- размер.

Для еще не созданных:
- карточка участника;
- имя и фамилия;
- фото или нейтральный плейсхолдер;
- подпись "Ожидаем создателя".

Сетка строится по заранее загруженному списку 29 участников, а не по абстрактным пустым слотам.

## 8. Повторная отправка

MVP-правило: один участник может создать только одну футболку.

Механика:
- участник выбирает себя из списка и подтверждает по фото;
- перед переходом к форме сайт проверяет статус участника в базе;
- если у участника уже есть заявка, форма блокируется и показывается финальная сцена;
- после успешной отправки заявка привязывается к `participant_id`;
- в базе должен быть уникальный запрет на повторную заявку для одного `participant_id`;
- `localStorage` можно использовать как быстрый способ сразу открыть финальный экран в том же браузере.

Ограничение:
- без персональных ссылок другой человек теоретически может выбрать чужое имя, если узнает участника по фото.

Принятое решение:
- сайт распространяется одной общей ссылкой в групповом чате;
- уникальные invite-ссылки не используются;
- основной антидубль работает через `participant_id` и статус в базе.

## 9. UX и визуальный стиль

Общее направление:
- mobile-first: сначала проектируем телефонный экран, затем расширяем до tablet/desktop;
- темная минималистичная тема;
- ощущение студии дизайна, а не Google Form;
- чистые крупные состояния;
- выразительный мокап футболки как главный объект;
- плавные переходы между шагами.

Визуальные принципы:
- фон: почти черный, глубокий графит;
- акценты: белый, серебристый, один яркий фирменный акцент;
- типографика: крупные заголовки, плотная композиция, хорошая читаемость;
- UI: компактные кнопки, четкие инпуты, без декоративного шума.

Mobile-first правила:
- основные действия доступны большим пальцем без точного попадания;
- кнопки и варианты выбора имеют комфортную touch-зону;
- мокап футболки остается главным объектом, но не вытесняет форму ниже первого экрана;
- каждый шаг должен помещаться в мобильный экран без ощущения перегруза;
- клавиатура телефона не должна ломать ввод имени и 3 слов;
- длинные имена и слова не должны вызывать горизонтальный скролл.

Анимации:
- появление шагов через fade/slide;
- микроанимации выбора размера;
- плавное обновление текста на футболке;
- финальная анимация отправки футболки в сетку;
- сетка обновляется без перезагрузки, если используется realtime.

Важно:
- анимации не должны мешать заполнению;
- весь флоу должен быть удобен на мобильном экране до desktop-полировки.

Распознавание участника:
- участник может писать имя на русском, украинском или английском;
- допускаются небольшие опечатки;
- поиск должен использовать нормализацию строк и fuzzy-ranking;
- перед переходом дальше участник всегда подтверждает найденную карточку.

## 10. Данные

Источник списка участников:
- Google Sheet: "Взносы на дни рождения 2026";
- вкладка: "Взносы";
- диапазон: первый столбец `A`;
- исключены из проекта: "Андрей Троян" и "Віталій Максимук";
- итоговое количество участников: 29.

Сущность `submission`:
- `id`
- `created_at`
- `participant_id`
- `first_name`
- `last_name`
- `size`
- `word_1`
- `word_2`
- `word_3`
- `client_submission_id`

Сущность `participant`:
- `id`
- `display_name`
- `first_name`
- `last_name`
- `search_name`
- `photo_file_name`
- `photo_url`
- `status`
- `created_at`
- `updated_at`

Фото:
- реальные фото будут добавлены позже в `public/participants/photos`;
- MVP использует плейсхолдеры, если фото участника отсутствует;
- рекомендуемое имя файла фото совпадает с полным именем участника, например `Альона Дейниченко.jpg`.

## 11. Админ/экспорт

MVP-минимум:
- данные доступны в Supabase Table Editor;
- можно экспортировать CSV.

Принятое решение:
- отдельная админка в MVP не нужна;
- хранение данных делаем в Supabase;
- итоговые данные для полиграфии выгружаются из Supabase в CSV.

## 12. Критерии успеха MVP

- Участник может пройти весь флоу от старта до подтверждения.
- Участник может найти себя по имени/фамилии и подтвердить себя по фото.
- Если участник уже создал футболку, форма для него не открывается.
- Все обязательные поля валидируются.
- Персонализация сразу отображается на мокапе.
- После подтверждения данные сохраняются в базе.
- После подтверждения повторный вход показывает финальную сцену, а не форму.
- Общая сетка отображает всех 29 участников и их статусы.
- Мобильная версия является основной и проходит проверку первой.
- Desktop-версия аккуратно расширяет мобильную композицию, а не задает ее.
- Есть простой способ получить итоговые данные для заказа футболок.

## 13. Риски

- Без уникальных ссылок человек технически может выбрать чужого участника, если сделает это намеренно.
- Слишком длинные слова или имена могут ломать композицию мокапа.
- Если финальный макет печати требует точных размеров, экранный мокап не заменяет подготовку print-файла.
- Realtime-сетка требует корректной настройки доступа к базе.
- Для фичи узнавания нужны фото всех участников в подходящем качестве.

## 14. Открытые вопросы

2. Нужно ли собирать дополнительные данные: телефон, Telegram, комментарий, предпочтение мужской/женской посадки?
3. Нужно ли показывать только перед футболки или перед и спину?
4. В каком формате потом нужен экспорт для производителя: CSV, Google Sheets, PDF, PNG-мокапы?
5. Есть ли дедлайн запуска?
6. Где будет размещаться сайт: Vercel, личный домен, временная ссылка?
